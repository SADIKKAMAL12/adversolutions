# Deploy AdverSolutions to Vercel + Supabase

## Prerequisites
- Node.js 18+ installed
- A Vercel account (free at https://vercel.com)
- A Supabase project (already created)

## Step 1: Set up Supabase Database

1. Go to your Supabase project dashboard: https://supabase.com/dashboard/project/srgfvbopyxuolyazasjz
2. Open the **SQL Editor**
3. Create a **New Query**
4. Copy and paste the entire contents of `supabase_schema.sql` from this project
5. Click **Run**

This creates all tables, seed data, and RLS policies.

## Step 2: Deploy to Vercel

### Option A: CLI Deployment (Recommended)

```bash
# Login to Vercel
npx vercel login

# Deploy (follow prompts)
npx vercel

# For production deployment
npx vercel --prod
```

### Option B: Git Deployment

1. Push this project to a GitHub/GitLab/Bitbucket repository
2. Go to https://vercel.com/new
3. Import your repository
4. Vercel will auto-detect Vite and build settings

## Architecture

- **Frontend**: Vite + React 18, deployed as static site
- **Backend**: Supabase (PostgreSQL + REST API)
- **Serverless Functions**: Vercel API routes for admin operations requiring service role key
- **Auth**: Custom email/password with bcrypt hashing, session stored in localStorage
- **Data Flow**: App hydrates from Supabase on load, writes key operations back to Supabase

## Troubleshooting

**Build fails?**
- Run `npm install` to ensure all dependencies are installed
- Check that `vite.config.js` exists

**Supabase connection fails?**
- Verify env vars are set correctly in Vercel dashboard
- Check browser console for CORS errors
- Ensure RLS policies allow the operations you're testing

**Blank page after deploy?**
- Check browser console for JS errors
- Verify `vercel.json` rewrites are configured
- Ensure `dist/` folder is generated by build
